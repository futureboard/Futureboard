# Getting Started

## First launch

When Studio starts it shows the **Welcome window** before the main workspace.
From here you can:

- **Create a new project** — choose a template and location.
- **Open an existing project** — browse for a `.mochiproj` file.
- **Open a recent project** — pick from the recent list.

Selecting a project opens a short **Loading Session** dialog while the audio
engine is built and warmed up, then hands off to the main Studio window.

## Audio setup

Open **Settings ▸ Audio** to choose your output device and driver. Studio
supports low-latency backends (including WDM-KS/WaveRT on Windows). Set the
buffer size to balance latency against CPU load:

- Smaller buffers → lower latency, higher CPU.
- Larger buffers → higher latency, more headroom.

The driver status line reports the active device and sample rate. Use
**Details** for the full device report.

## Creating a project

1. In the Welcome window, choose **New Project**.
2. Pick a location and name. Projects save as `.mochiproj`.
3. The main workspace opens with an empty arrangement.

## Adding your first track

Use **Track ▸ Add** (or right-click the track area) and pick a track type:

- **Audio** — record or import audio.
- **MIDI** — sequence notes for a plugin instrument.
- **Instrument** — a MIDI track with a hosted instrument on its input.
- **Bus / Group / Return / Master** — routing and summing.

See [Tracks & Clips](./tracks-and-clips.mdx) for details.

## Saving

- **File ▸ Save** writes the current project.
- **File ▸ Save As** writes a copy.

Project save/load persists your arrangement, mixer state, routing, automation,
**and hosted plugin state**, so a reopened project restores exactly.
